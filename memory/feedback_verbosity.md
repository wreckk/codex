---
name: Verbose thinking
description: User wants more explanation of what is being tried and why during debugging
type: feedback
---

Be more verbose when debugging or iterating — explain what hypothesis you're testing, why the previous attempt failed, and what the new approach does differently.

**Why:** User wants to follow along and understand what's changing, not just see silent code edits.

**How to apply:** Before each code change, write 2–3 sentences explaining the root cause diagnosis and why the new approach addresses it.
